このリポジトリは、Xアカウントを運用することを目的としている。

## 構成
- アカウントごとに `accounts/` 配下でディレクトリを分ける（例: `accounts/agent-x/`）。
- 各アカウントは、AI Agent がそのアカウント配下の `AGENT_PROMPT.md` に従って `workspace/` を読み書きし、下書き作成と学習ループを回す。

## 対話型（人間とAIの協調）
- 人間との対話はチャットで行う。`<account>/workspace/human/*` は自律運用のI/O専用（「自律運用に渡す用件」を置く用途に限定）。
- 対話型セッションは、このリポジトリ全体をメタ的に改善するために使う（設計、運用方針、プロンプト、ディレクトリ構成、実行スクリプト、ガードレールなど）。
- アカウント固有の作業は必ず対象アカウントのディレクトリを明示して行う（混線させない）。

## 目標
- Xアカウント運用で収益化を目指す。
- AIっぽい文章を避ける。無難な一般論・テンプレ口調・過剰な箇条書きを避ける。
- デフォルトは人間レビュー前提。投稿などの対外アクションは、明示的な承認がある場合のみ行う。
  - 例外: 人間が `AUTO_PUBLISH=1` を明示設定したアカウントは、ガードレール付きの自動投稿を許可する（kill switch / rate limit / duplicate prevention）。

## ディレクトリ規約
- アカウント実体: `accounts/<account>/`（例: `accounts/agent-x/`）
- 自律運用プロンプト: `<account>/AGENT_PROMPT.md`
- 運用データ: `<account>/workspace/`
  - 下書き、メモリ、状態、ログ、人間との非同期メッセージなど「運用で増えるデータ」を置く場所。
  - `workspace/human/requests.md`: Agent -> 人間（依頼）
  - `workspace/human/messages.md`: 人間 -> Agent（返答）
  - `workspace/human/*` は「自律運用のI/O」として予約する。対話型セッションの会話ログ置き場としては使わない（明示指示がある場合のみ読み書きする）。

## 変更の制約
- 指示がない限り「ツール/スクリプトを増やす」方向で問題を解かない（運用設計とプロンプト改善を優先）。
- リポジトリ改善の変更は必要最小限にし、意図と影響を明確にする。
- シークレットはコミットしない。
  - 推奨: 環境変数（またはサーバ側の秘密管理）で注入する。
  - 例外としてローカルファイルで管理する場合は、`<account>/workspace/secrets/` のような gitignore 対象ディレクトリ配下に置く。
